## APILimiter - Защита от спама вызовов к API
*Команд: 2*

### Команды

> [!command]  
**`.api_fw_protection`**
  Включает или отключает защиту API от флуда. Открывает инлайн с подтверждением. Когда защита активна - модуль перехватывает все исходящие запросы и замедляет их, если порог превышен.

> [!command]  
**`.suspend_api_protect`** `<секунды>`
  Временно приостанавливает защиту от флуда(api_fw) на указанное количество секунд.


####  Конфиг `(.cfg APILimiter)`
| Ключ                | Описание                                                                    |
| :------------------ | :-------------------------------------------------------------------------- |
| `time_sample`       | Время в секундах для подсчёта запросов (по умолчанию: 15)                   |
| `threshold`         | Максимальное количество запросов до срабатывания защиты (по умолчанию: 100) |
| `local_floodwait`   | Пауза в секундах при срабатывании (от 10 до 3600, по умолчанию: 30)         |
| `forbidden_methods` | Список запрещённых методов API.                                             |

## Evaluator - Выполнение кода на Python, C, C++, Node.js,GoLang,Rust прямо в телеграме
*Команд: 6*

### Команды

> [!command] Сommand (алиас: `.eval`)
**`.e`**  `<код>` 
Можно ответить на сообщение - его текст станет кодом(если код валидны).
  Выполняет питон-код в контексте юзербота. 
  Выводит результат. Автоматически цензурирует чувствительные данные (токены, session, номер телефона).

> [!command] 
> **`.ecpp`** `<код>` 
Компилирует и выполняет код на C++ (требует `g++`). Компилирует во временной директории, затем запускает.

> [!command] 
> **`.ec`**  `<код>`
  Компилирует и выполняет код на C (требует `gcc`). Аналог `.ecpp`, но для C.

> [!command] 
> **`.enode`** `<код>` 
  Выполняет JS-код через Node.js (требует `node`). Создаёт временный .js-файл и запускает его.

> [!command] 
> **`.eg`** `<код>` 
  Выполняет Go-код Создаёт временный файл и запускает его.
  
  > [!command] 
> **`.ers`** `<код>` 
  Выполняет Rust-код Создаёт временный файл и запускает его.

## Help
Показывает справку по модулям и командам
*Команд: 3*

### Команды

  > [!command] 
**`.help`** `[название]` -флаг
  Выводит список всех доступных модулей и команд. Без аргументов - общий список. С именем модуля или команды - подробная справка по конкретному модулю.
`-f` ==[флаг]== - Показать все команды, включая недоступные по правам.
`-c` ==[флаг]== - Показать только core-модули.
`-l` ==[флаг]== - Показать только загруженные внешние модули.

  > [!command] 
**`.helphide`** `<модуль ...>` - один или несколько модулей через пробел.
  Скрывает или показывает модули в общем списке `.help`. Если модуль уже скрыт - делает его видимым, и наоборот.

   > [!command] 
**`.support`** - Отправляет ссылку на официальный чат поддержки Heroku.

#### Конфиг `(.cfg Help)`
| Ключ                   | Описание                             |
| :--------------------- | :----------------------------------- |
| `core_emoji`           | Эмодзи для кор-модулей в списке      |
| `plain_emoji`          | Эмодзи для внешних модулей в списке  |
| `banner_url`           | ссылка на баннер, отображаемого в .help |
| `show_preview_in_help` | Показывать превью при .help [модуль] |

## HerokuBackup - Резервное копирование и восстановление базы данных и модулей
*Команд: 7*

### Команды

   > [!command] 
**`.backupall`**
  Создаёт бэкап ваших данных,а именно: база данных + все модули + pip-зависимости. Бэкап сохраняется в топик **Backups** в формате `.backup`

   > [!command] 
**`.backupdb`**
  Создаёт бэкап базы данных и отправляет файл в топик **Backups**

   > [!command] 
**`.backupmods`**
  Создаёт бэкап всех загруженных модулей в виде zip.

> [!command] 
**`.restoreall`** - **`Ответ на backup`**
  Восстанавливает бэкап из файла `.backup`. Необходимо ответить на сообщение с файлом бэкапа. После восстановления Heroku автоматически перезапускается.

> [!command] 
**`.restoredb`** - **`Ответ на json`**
  Восстанавливает базу данных.
  
> [!command] 
**`.restoremods`** - **`Ответ на zip`**
  Восстанавливает модули.

> [!command] 
**`.set_backup_period`** **`<часы>`**  (число от 0 до 199)
  Устанавливает период автоматического резервного копирования.

## HerokuConfig
*Команд: 2*

### Команды

> [!command] Commands (алиас: `.cfg`)
 **`.config`* `[модуль] [флаг]` - Название модуля и конкретного параметра для быстрого 
  Открывает конфигуратор модулей через инлайн.

> [!command] Commands (алиас: `.fcfg`)
**`.fconfig`** `<модуль> <флаг> <значение>` - Установить значение параметра. 
  Изменение конфига без инлайна. 
Есть разделитель `&&`. Если значение не указано - берётся текст реплая.
Пример: `.fconfig Help banner_url https://...`
`<флаг> <значение> && <флаг> <значение>` -  Несколько параметров через `&&`.

## HerokuSecurity - Управление правами доступа к модулям / командам
*Команд: 14*

### Команды

> [!command]  
**`.security`** `[команда]`
  Открывает меню управления правами.
   Без аргументов - инлайн меню. 
   С именем команды - права для конкретной команды.

> [!command]  
**`.inlinesec`** `[инлайн-команда]`
  Аналог `.security`, но для инлайн-команд бота.

> [!command]  
**`.querysec`**
  Управляет разрешением использовать инлайн-запросы (@ бота в любом чате).

> [!command]  
**`.owneradd`** `<@user> / <id> / <reply>`
  Даёт полные права на управление юзерботом.
`-f / --force` ==[флаг]== - Добавить без подтверждения.
`-n / --nonick` ==[флаг]== - Также добавить в nonick-список.

> [!command]  
**`.ownerrm`** `<@user> / <id> / <reply>`
  Удаляет пользователя из списка owner. 

> [!command]  
**`.ownerlist`**
  Показывает список всех пользователей с доступом к юб.

> [!command]  
**`.newsgroup`** `<имя>`
  Создаёт новую security group - именованную группу для управления правами.

> [!command]  
**`.sgroups`**
  Показывает список всех security groups с числом участников и правил в каждой.

> [!command]  
**`.sgroup`** `<имя группы>`
  Показывает подробную информацию о конкретной security group: участники и их права.

> [!command]  
**`.delsgroup`** `<имя группы>`
  Удаляет security group вместе со всеми её правилами и участниками.

> [!command]  
**`.sgroupadd`** `<группа> <@user> / <id> / <reply>`
  Добавляет пользователя в security group. 

> [!command]  
**`.sgroupdel`** `<группа> <@user> / <id> / <reply>`
  Удаляет пользователя из security group.

> [!command]  
**`.tsec`** `<команда / модуль>` `[@user/chat]`
  Временно выдаёт право на выполнение команды или модуля пользователю или чату.
 Можно указать длительность.
 Например: `1d`, `2h`, `30m`, `60s`. Без указания - навсегда.

> [!command]  
**`.tsecrm`** `<sec-правило>`
  Удаляет конкретное временное правило доступа.

> [!command]  
**`.tsecclr`**
  Очищает все временные правила доступа сразу.
